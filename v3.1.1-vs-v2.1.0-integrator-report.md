# Ekubo v3.1.1 vs v2.1.0 Integrator Report

This report compares tag `v2.1.0` (`79ae633c5b8916636bab229e92ad3131a44f00cd`) to tag `v3.1.1`
(`f8e38b5be3f856ff99d98475be4f10af7f593a58`).

The ABI data below was generated from compiled artifacts after running `forge build --offline` in clean tag snapshots.
Both snapshots compiled successfully. v2.1.0 compiled 130 Solidity files with solc `0.8.28`; v3.1.1 compiled 226
Solidity files with solc `0.8.33`.

## High Impact Migration Items

1. Core position events changed topic0 and data layout.
   `PositionUpdated` and `PositionFeesCollected` are not backward-compatible. Indexers must decode the v3 formats
   separately from v2.

2. Core no longer exposes ownership or protocol-fee withdrawal ABI.
   `owner`, ownership handover functions/events, and `withdrawProtocolFees(address,address,uint256)` were removed from
   Core. Ownership and protocol fee withdrawal now exist on Positions-derived contracts.

3. Core saved-balance and payment APIs were redesigned.
   v2 `load`, `save`, `pay(address)`, and `withdraw(address,address,uint128)` were removed. v3 adds
   `updateSavedBalances`, `startPayments`, `completePayments`, `withdraw()`, and `updateDebt()`.

4. Position identity changed.
   v2 represented a core position as `PositionKey { salt, owner, Bounds { lower, upper } }`. v3 uses a packed
   `PositionId` (`bytes32`) with `bytes24 salt`, `int32 tickLower`, and `int32 tickUpper`. In the Positions contract,
   the salt is derived from the NFT id as `bytes24(uint192(id))`.

5. Positions user-facing bounds changed from a tuple to flat ticks.
   Calls such as `deposit`, `withdraw`, `collectFees`, `mintAndDeposit`, and `getPositionFeesAndLiquidity` replaced
   `(int32,int32) bounds` with separate `int32 tickLower, int32 tickUpper`, changing selectors.

6. Pool config is still ABI-encoded as `bytes32`, but the packed semantics changed.
   v2 `Config` packed `extension | fee | tickSpacing`. v3 `PoolConfig` packs `extension | fee | typeConfig`, where
   `typeConfig` supports concentrated pools and stableswap pools. Integrators that construct `PoolKey.config` must use
   the v3 helpers/packing.

7. Deployment arguments changed.
   v2 Core took an `owner` constructor argument; v3 Core has no ABI constructor. v2 Positions took
   `(core, tokenURIGenerator)`; v3 Positions takes `(core, owner, swapProtocolFeeX64,
   withdrawalProtocolFeeDenominator)`.

### Core Event Format Notes

None of the Core position event fields are indexed in either version, so only `topic0` is present.

`PositionFeesCollected` data layout changed from:

```text
v2 data words:
poolId, positionKey.salt, positionKey.owner, positionKey.bounds.lower, positionKey.bounds.upper, amount0, amount1
```

to:

```text
v3 data words:
locker, poolId, positionId, amount0, amount1
```

`PositionUpdated` data layout changed from:

```text
v2 data words:
locker, poolId, params.salt, params.bounds.lower, params.bounds.upper, params.liquidityDelta, delta0, delta1
```

to:

```text
v3 data words:
locker, poolId, positionId, liquidityDelta, balanceUpdate, stateAfter
```

In v3, `balanceUpdate` is a packed `bytes32` value containing the pool token deltas, and `stateAfter` is the packed
post-update pool state. Indexers that previously read `delta0` and `delta1` as separate ABI words must decode
`PoolBalanceUpdate` instead.

## Integrator Checklist

- Update Core event indexers to branch by contract version or topic0 for the new v3 `PositionUpdated` and
  `PositionFeesCollected` events.
- Stop decoding Core ownership and protocol-fee events in v3; track ownership on Positions/manager contracts instead.
- Replace v2 `PositionKey` derivation with v3 `PositionId` packing.
- Replace Positions calls that pass `Bounds` tuples with flat `tickLower` and `tickUpper` arguments.
- Replace direct Core saved-balance/payment flows with `updateSavedBalances` or the v3 `startPayments` /
  `completePayments` / packed `withdraw()` flow.
- Rebuild any extension integrations against the v3 `IExtension` hooks, because update-position and swap hooks now pass
  typed packed state such as `PositionId`, `SwapParameters`, `PoolBalanceUpdate`, and `PoolState`.
- Recreate `PoolKey.config` with v3 `PoolConfig` helpers. Do not reuse v2 tick-spacing packing for new v3 pools.
